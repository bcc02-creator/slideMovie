## 1. 簡介

**Slidecast** 是一款純瀏覽器內運行的工具，讓你把投影片（PDF、圖片或 HTML）搭配配音和字幕，直接合成為可下載的影片。**不需要安裝任何軟體，也不需要後端伺服器**，所有處理都在你的瀏覽器中完成。

---

## 2. 系統需求

- **瀏覽器**：Chrome 或 Edge（最新版，建議使用）
- Firefox / Safari 可以預覽，但 MP4 轉檔可能受限
- **硬碟空間**：瀏覽器需要足夠的 IndexedDB 空間存放暫存檔案
- **麥克風**：不需要，配音由上傳音檔或文字轉語音（TTS）產生

---

## 3. 介面概覽

介面分為左右兩欄：

| 區域 | 說明 |
|------|------|
| **左側面板** | 設定區：依序完成各步驟的資料上傳與設定 |
| **右側預覽** | Canvas 即時預覽目前投影片與字幕疊層 |
| **底部工具列** | 播放控制、錄製、下載、專案儲存 |
| **右上角** | 解析度設定、字幕開關 |

## 4. 步驟一：載入投影片

支援三種來源，**選擇其中一種即可**：

### 4.1 PDF
拖曳 `.pdf` 檔案，或點擊上傳區。
- 系統會自動逐頁轉成圖片，轉換期間顯示進度條。
- 支援錄製。

### 4.2 圖片
拖曳多張圖片（`.jpg`、`.png`、`.webp` 等），**可一次選取多張**，會依選取順序排列成投影片。
- 支援錄製。

### 4.3 HTML Deck
拖曳單一 `.html` 檔案（適用於 Reveal.js、Deck.js 等自製 HTML 投影片）。
- **僅支援預覽，無法錄製**（瀏覽器安全限制）。

> **提示**：投影片載入後，右側預覽會立即顯示第一張。

---

## 5. 步驟二：設定配音

有兩種模式，切換後設定不同：

### 5.1 上傳音檔（預設）
- 拖曳一或多個音訊檔案（`.mp3`、`.wav`、`.m4a`、`.ogg` 等）
- **每個音檔對應一張投影片**（第 1 個音檔 → 第 1 張，以此類推）
- 音檔數量與投影片數量不一致時，介面會顯示黃色警告

### 5.2 文字轉語音（TTS）
1. 切換到「TTS 模式」
2. 需要同時上傳**字幕檔**（見步驟三），字幕的每行文字就是要朗讀的內容
3. 可選擇語音（優先列出中文語音）、語速、音調
4. **TTS 模式只能即時預覽，無法錄製成影片**

---

## 6. 步驟三：載入字幕（選用）

字幕讓影片在播放時疊加文字。支援格式：

| 格式 | 副檔名 | 說明 |
|------|--------|------|
| SRT | `.srt` | 標準時間軸字幕 |
| WebVTT | `.vtt` | 網頁字幕格式 |
| JSON | `.json` | 自訂格式（`[{start, end, text}]`） |
| 純文字 | `.txt` | 每行一句，依段落自動分配到每張投影片 |

> 如果使用 TTS 模式，字幕文字同時也是 TTS 朗讀的內容，**此時字幕為必填**。

---

## 7. 步驟四：背景音樂（選用）

- 上傳音樂檔案（`.mp3`、`.wav` 等），播放時自動混音到配音背景
- **音量滑桿**：預設 18%，可依喜好調整
- **試聽按鈕**：可在不播放投影片的情況下單獨試聽背景音樂

---

## 8. 步驟五：片頭／片尾（選用）

在影片最前或最後加入過場素材：

| 欄位 | 說明 |
|------|------|
| 片頭 / 片尾 | 上傳圖片（`.jpg`/`.png`）或影片（`.mp4`/`.webm`） |
| 持續時間 | 圖片型片頭/片尾的顯示秒數（預設 3 秒） |
| 影片型 | 播到結束為止，忽略持續時間設定 |

> 影片型片頭/片尾的**音訊會被錄製進去**。

---

## 9. 預覽播放

底部工具列控制項：

| 按鈕 | 說明 |
|------|------|
| ▶ 播放 | 從目前位置開始播放 |
| ⏸ 暫停 | 暫停播放，保留目前位置 |
| ⏹ 停止 | 停止並回到開頭 |
| 進度條 | 點擊或拖曳可跳轉位置（TTS 模式跳到最近的段落起點）|

**右上角**：
- **字幕開關**：切換字幕疊層顯示
- **解析度**：選擇錄製解析度（預設 1920×1080）

## 10. 錄製影片

### 前置條件（缺一不可）
- 投影片來源為 **PDF 或圖片**（HTML Deck 不支援錄製）
- 已載入至少一段**上傳的配音音檔**（TTS 模式不支援錄製）

### 錄製流程
1. 點擊 **「開始錄製」** 按鈕
2. 系統自動依序執行：
   - **片頭**（如有設定）
   - **主體**：從第一張投影片自動播放到結尾
   - **片尾**（如有設定）
3. 錄製完成後顯示通知音效與完成提示

### 中途中止
- 點擊 **「停止錄製」** 可隨時中斷，已錄製的部分仍可下載

---

## 11. 下載影片

錄製完成後出現兩個下載選項：

### WebM（直接下載）
- 瀏覽器原生格式，立即可用
- 副檔名：`slidecast-<時間戳>.webm`

### MP4（轉檔後下載）
- 透過 **ffmpeg.wasm** 在瀏覽器內轉檔，不需上傳到伺服器
- 轉檔時顯示進度條，完成前請勿關閉頁面
- **需要 COOP+COEP HTTP Headers 支援**（本機開發與 Vercel 部署均支援；GitHub Pages 不支援）

---

## 12. 專案管理

### 儲存專案
- **Cmd/Ctrl + S**：手動儲存
- **自動儲存**：編輯後 4 秒自動儲存（**僅限手動儲存過至少一次的專案**）
- 所有資料存在瀏覽器 **IndexedDB**（本機，不上傳伺服器）

### 開啟/管理專案
- 點擊工具列的 **「專案」** 按鈕開啟清單
- 可重新命名或刪除舊專案
- 可查看各專案的儲存空間用量

> **注意**：清除瀏覽器資料或 IndexedDB 會刪除所有已儲存的專案。

## 13. 設定說明

| 設定 | 預設值 | 說明 |
|------|--------|------|
| 解析度 | 1920×1080 | 影片輸出解析度（錄製前設定） |
| 字幕顯示 | 開 | 是否在預覽/錄製中顯示字幕疊層 |
| 背景音樂音量 | 18% | BGM 相對於配音的音量比例 |
| TTS 語速 | 1.0 | 1.0 為正常速度，0.5 慢速，2.0 快速 |
| TTS 音調 | 1.0 | 調整語音音調高低 |
| 自動儲存 | 開 | 編輯後 4 秒自動儲存（需先手動儲存一次） |

## 14. 常見問題

**Q：PDF 解析很慢怎麼辦？**
A：正常現象，頁數越多越久。解析期間可先設定其他選項。

**Q：投影片數量和音檔數量不一樣？**
A：介面會顯示警告，但仍可播放。建議保持相同數量（一張投影片一個音檔）。

**Q：MP4 按鈕是灰色的？**
A：表示當前環境不支援 SharedArrayBuffer（例如 GitHub Pages），請改用 WebM 或換到支援的部署環境。

**Q：TTS 的聲音怎麼沒辦法錄製？**
A：瀏覽器 Web Speech API 的輸出無法被 MediaRecorder 捕捉，這是瀏覽器本身的安全限制。若需要錄製，請改用上傳音檔模式。

**Q：專案儲存在哪裡？**
A：存在你瀏覽器的 IndexedDB，不會上傳到任何伺服器。換電腦或清除瀏覽器資料後會消失。

**Q：如何讓投影片順序正確？**
A：上傳圖片時，瀏覽器通常依檔名排序。建議圖片命名為 `01.jpg`、`02.jpg` 等確保順序正確。

| 功能 | PDF/圖片 | HTML Deck |
|------|----------|-----------|
| 預覽播放 | ✓ | ✓ |
| 字幕疊層 | ✓ | ✓ |
| 錄製影片 | ✓ | ✗ |
| TTS 預覽 | ✓ | ✓ |
| TTS 錄製 | ✗ | ✗ |
| MP4 轉檔 | 需 COEP/COOP | 需 COEP/COOP |
